# Примечания к реализации:
* Если данные должны использоваться более чем одним компонентом, то они хранятся в Redux.
* Если данные должны использоваться только одним компонентом, то они могут храниться в локальном стейте.
* Для передачи функции вместе со своим контекстом в таймер используется классовый компонент. Во всех остальных случаях используются функциональные компоненты.
* Реализован простой адаптивный дизайн.
